BST Analysis Toolkit

As a final project for Special Topics in Algorithms, Graphs, and Data Structures in spring of 2017, a friend and I created a visualization toolkit and analysis tool for binary search trees. We implemented Red-Black, Splay, AVL, WAVL, and simple BSTs using an API we wrote for interfacing with a tree in a way that records every access, modification, addition, and deletion. This is based on research by John Iacono on the geometric interpretation of binary search trees. The project, along with detailed documentation and more disucssion of the theory, can be found on GitHub.

SandDial

For a fun week's project over the summer of 2017, and to teach my younger brother programming, we built what we call a Sand Dial — a clock built from a self-turning hourglass. The basic mechanism is an hourglass stuck to a servo, across from a camera. The camera and motor are both run by a Raspberry Pi. Using OpenCV, we made a tight routine of [thresholding -> Canny edge detection -> edge dilation -> contour separation to find the location of the sand in the image. When no sand is detected in the top of the hourglass, a signal is sent to the servo to flip itself. A video of it in action can be found here. My code can be found on GitHub here and, for posterity, my brother's can be found here.

CardControl

CardControl is a web application for access control in a University system. It was built as a semester project for a Web Engineering class in spring of 2017, with one other friend. It is built on the Angular 2 frontend framework, with a backend of Django, PostgreSQL, NGINX, Varnish, Redis, and AWS. I worked largely on those backend components, while the other student created most of the frontend. I also wrote an API using Tastypie for automated access control. We implemented many server-side infrastructure niceties, like CI through webhooks, simplistic process monitoring, and automated installation of all dependencies. To show off our access control application, we built a second web application (A Piazza clone named Azzaip) on a similar stack which used the API for our system to control which users had access to which site resources. The applications are not running currently (Running multiple AWS instances is expensive for a student!) but I hope to run them off of this site's server eventually, time permitting. Regardless, the code for CardControl can be found here and for Azzaip here.

Turing Machine

For one homework assignment for Tufts' Computational Complexity Theory class in the spring of 2017, we had to write a Turing machine by hand, which the grading staff would then run through an interpreter. I took it upon myself as an independent project after the assignment to write a greatly improved interpreter for the Turing machine's syntax, which is this project. It has three debug modes corresponding to three different UIs, supports step-through debugging, and an animation mode to see the tape change in real time. It is written in Python. More can be found on its GitHub page, which is here.
